Part 02
Hadoop, Spark & Big Data Technologies

Hadoop was a revolution — one framework for both storing and processing petabytes of data. Spark made it even faster with in-memory computation.

🐘 Hadoop

Storage + Processing

A framework that handles both huge data storage and huge data processing — unlike traditional databases that only store, or C/Java that only process.

  • HDFS — Hadoop Distributed File System (storage)
  • MapReduce — original processing engine (now legacy)
  • Spark — modern replacement for MapReduce
  • Supports TB/PB scale data across many machines
  • Available on AWS, Azure, GCP cloud platforms
⚡ Apache Spark

In-Memory Computation

Spark is the "hot cake" of big data today — a very high-speed processing execution model.

  • Storage and processing happen in RAM, not disk
  • Result: dramatically faster than disk-based systems
  • Parallel processing across multiple machines simultaneously
  • Used for both batch and real-time stream processing
  • The engine powering modern ELT pipelines
Technology Role Key Capability
Oracle / MySQL / SQL Server Traditional DB storage Structured data only, TBs limit Limited
C / C++ / Java Processing only No distributed storage capability
Hadoop (HDFS) Distributed storage Petabyte-scale data across many nodes Scalable
Apache Spark In-memory processing High-speed parallel computation Fast
AWS / Azure / GCP Cloud platforms Managed Hadoop & Spark jobs at scale

ETL → ELT: How Data Pipelines Evolved

When Hadoop changed how data is stored, the traditional Extract–Transform–Load pattern flipped into Extract–Load–Transform.

TRADITIONAL ETL
📥 Extract
from sources
→
🔄 Transform
apply business logic
→
📤 Load
into target DB
MODERN ELT (with Hadoop / Spark)
📥 Extract
from all branches
→
📤 Load
into staging area (HDFS)
→
🔄 Transform
with Apache Spark

💡 Why the flip? With Hadoop, you can first pull all raw data from 10 cities / branches into a central staging area, then apply business logic with Spark. This is faster and more flexible — you keep all raw data and transform only what you need, when you need it.

🏢
Data Sources (Multi-Branch)
Pull data from Delhi, Mumbai, Chennai, Hyderabad branches simultaneously into a central staging area — a distributed store that can handle the load.
🗃️
Staging Area
A central raw data store (e.g. HDFS) where all un-transformed data lands first. No processing until all data is consolidated. Enables flexible, on-demand transformation.
🧪
Transform with Spark
Apply business logic, filters, aggregations and enrichments using Apache Spark's in-memory processing. Extract only the required fields for downstream analytics or ML models.
Part 04
Generative AI & Why Python Matters

Generative AI creates new content from learned patterns — and Python is the essential language for working with all of it.

🎨 Generative AI

Produces New Data

Instead of just analyzing or classifying existing data, generative AI produces new outputs — text, images, code, music — based on what it learned from training.

  • Trained on large datasets (text, image, audio)
  • Learns patterns, relationships, structures
  • Generates new content from user prompts
  • Built on Transformer architectures
  • Examples: GPT-4, Claude, Gemini, DALL·E, Sora
🔍 Discriminative AI

Classifies Existing Data

Traditional AI focused on analyzing, predicting, and classifying data that already exists — outputting a label, score, or decision.

  • Image recognition (identifies what's in a photo)
  • Sentiment analysis (positive/negative)
  • Fraud detection (fraud/not fraud)
  • Medical diagnosis classification
  • Examples: CNN, Logistic Regression, SVM
🐍

Why You Must Learn Python

People ask: "AI generates code for me — why learn Python?"

Python is the mother tongue of AI & Data Science. Just as you communicate with family in your mother tongue, you need Python to communicate with AI and ML technologies. If AI generates code and you don't know Python, you can't understand, debug, or modify what it produced. You need to know: is this a list? a function? a class? Python makes AI work faster — it doesn't replace the need to learn it.

Understand AI output Debug generated code Import library functions Work with data pipelines Build ML models
Part 05
Agentic AI — The Next Frontier

Generative AI creates content on demand. Agentic AI goes further — it plans, decides, and executes multi-step actions autonomously to achieve a goal.

🎨 Generative AI

Single Output

You prompt → it responds. One interaction, one output. Great for content creation but you're still in the driver's seat for every next step.

🤖 Agentic AI

Multi-Step Autonomous Execution

You give a goal → it plans, breaks it into tasks, executes series of actions, uses tools, and delivers results — with minimal human intervention.

✈️
EXAMPLE — VACATION PLANNING AGENT

You say: "Plan a trip to Goa for 5 days" — the agent does:

1
Search flights to the destination and compare prices
2
Compare hotel prices near tourist spots
3
Suggest tourist attractions relevant to your interests
4
Book reservations once you approve — all without you lifting a finger

Other real-world agentic use cases from the lecture:

🛎️
Customer Support Agent
Receives customer complaints → checks order status → processes refunds → updates records — all with minimal human intervention. Replaces the "press 1, press 2" approach.
💻
Software Development Agent
Reads requirements → generates the full code → runs tests → fixes bugs → delivers a working feature — a complete development loop autonomously.
📊
Data Analysis Agent
"Why did sales drop in May?" — the agent collects sales data, performs analysis, creates charts, identifies root causes, and generates a full report. Series of actions, one goal.
