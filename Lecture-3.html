Part 01
Evolution of Artificial Intelligence

AI didn't leap straight to ChatGPT — it evolved through decades of breakthroughs in hardware, data, and algorithms.

Early Days
Rule-Based & Symbolic Approaches
Systems like decision trees and symbolic programming followed hardcoded rules. No learning ability — every behaviour was explicitly programmed by engineers.
1990s – 2000s
Machine Learning Revolution
AI shifted from rule-based to learning from data. Key developments: statistical models, neural networks, data mining (extracting useful insights from raw data). Systems improved by observing patterns in existing data.
Big Data Era
Big Data + Distributed Computing
Massive data generation enabled a shift from monolithic computation (one system for all) to distributed / parallel processing (multiple systems sharing a single file and processing it simultaneously). GPUs and internet-scale computation arrived.
Deep Learning
Deep Learning Breakthroughs
Deep learning techniques powered: Image recognition · Speech recognition · Language translation · Self-driving technology. Although deep learning existed earlier, it only became mainstream after big-data and GPU availability.
2020+
Large Language Models (LLMs)
Modern AI systems can now generate text, create images, write code, and compose music on their own. LLMs like GPT, Gemini, and Claude became mainstream from 2020 onwards, powered by the Transformer architecture.
Today
Agentic AI
The next frontier: AI agents that can autonomously plan, decide, and execute multi-step tasks — combining generation with reasoning and tool use. Still evolving rapidly.

Part 03
Generative vs Discriminative AI

Two fundamentally different paradigms — one creates, the other classifies.

🎨 Generative AI

Creates New Data

Purpose: generates new content based on a learned understanding of patterns.

  • Output: Text · Image · Audio · Code · Video
  • Example input: "Create a marketing email for a new smartphone launch"
  • Example output: Full email — subject line, body, call-to-action — generated
  • ChatGPT · DALL·E · Stable Diffusion · Gemini
🔍 Discriminative AI

Classifies Existing Data

Purpose: predicts, labels, or categorises data that already exists.

  • Output: Labels · Categories · Predictions · Scores
  • Example input: An email fed to a spam classifier
  • Example output: "Spam" or "Not Spam" — binary classification
  • Logistic Regression · Decision Tree · CNN

Part 05
Key Transformer Model Families

All modern generative AI is built on the Transformer architecture. Three major variant families serve different purposes.

🔵 BERT

Bidirectional Understanding

Bidirectional Encoder Representations from Transformers. Reads text in both directions to understand the full context of a word.

  • Sentiment analysis
  • Question answering
  • Text classification
Example: "The target is near the river." — BERT uses surrounding words to understand the meaning of "target" (a place, not a goal).
🟣 GPT Family

Content Generation

Generative Pre-trained Transformer. Generates text by predicting the next token, left-to-right, enabling fluid, coherent long-form output.

  • Content generation
  • Coding assistance
  • Chatbots & assistants
Examples: GPT-3/4/4o/o4-mini, ChatGPT (OpenAI), Claude (Anthropic), Gemini (Google)
🟢 T5

Text-to-Text Transfer

Text-to-Text Transfer Transformer. Frames every NLP task as a text generation problem — input text in, output text out.

  • Translation (English → Hindi / French)
  • Summarisation
  • Question answering
Key idea: Even classification is reformulated as "classify: [text]" → "positive" (text in, text out).

📄 Unimodal

Single Data Type

Works with only one modality of data at a time.

  • Text-only: classic language models
  • Image-only: CNNs for vision
  • Audio-only: speech recognition systems
🌐 Multimodal

Multiple Data Types

Combines information from different sources simultaneously for richer understanding.

  • Medical diagnosis: MRI scan + patient blood report → prediction & explanation
  • Video captioning: video frames + audio → text description
  • Visual Q&A: image + question → answer

🏥 Medical Diagnosis Example (from lecture):

Input: MRI scan (image) + Patient blood report (text)
Output: Prediction (diagnosis) + Explanation report

A multimodal AI can combine visual imaging data with structured medical records simultaneously — something a unimodal model simply cannot do.

🚀

The Big Picture

AI evolved from rigid rule-based systems → to statistical machine learning → to deep learning → to Generative AI. Unlike traditional AI that analyzes and classifies, Generative AI creates new content — text, images, code, audio, and video — based on patterns learned from massive datasets. The five model types (text, image, audio, video, code) each serve distinct use cases, while multimodal models combine all these capabilities into unified systems that mirror how humans naturally perceive and interact with the world.
